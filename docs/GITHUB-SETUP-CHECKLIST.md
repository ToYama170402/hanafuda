# GitHub セットアップチェックリスト

このドキュメントは、実装を開始する前にGitHub上で行うべき準備作業のチェックリストです。

## ✅ 準備作業の概要

実装を開始する前に、以下の3つの作業をGitHub上で完了させる必要があります：

1. **マイルストーンの作成**（4個）
2. **ラベルの作成**（約20個）
3. **Issueの作成**（30個）

---

## 📋 チェックリスト

### ステップ1: マイルストーンの作成

GitHubリポジトリの「Issues」タブ → 「Milestones」から作成

- [ ] **M1: MVP完成**
  - タイトル: `M1: MVP完成`
  - 期限: プロジェクト開始から3週間後
  - 説明: 基本的なゲームフローが動作する最小限の機能

- [ ] **M2: 完全ルール実装**
  - タイトル: `M2: 完全ルール実装`
  - 期限: プロジェクト開始から5週間後
  - 説明: こいこいを含む完全なゲームルールの実装

- [ ] **M3: 拡張機能実装**
  - タイトル: `M3: 拡張機能実装`
  - 期限: プロジェクト開始から6週間後
  - 説明: カスタマイズ機能やイベントシステムの実装

- [ ] **M4: リリース準備完了**
  - タイトル: `M4: リリース準備完了`
  - 期限: プロジェクト開始から8週間後
  - 説明: 最適化、ドキュメント、リプレイ機能の実装

## 📝 Issue作成のヒント

### Issueのテンプレート

各Issueを作成する際は、[issues-summary.md](issues-summary.md)のテンプレートを使用してください：

```markdown
## 機能の概要
[実装計画書の該当セクションから概要を記載]

## タスク内容
- [ ] タスク1
- [ ] タスク2
- [ ] タスク3

## 成果物
- ファイル名やクラス名
- テストケース

## 推定工数
X-Y時間

## 依存関係
Depends on #XX

## 参考資料
- [実装仕様書](../docs/implementation-specification.md)
- [ゲームルール仕様書](../docs/hanafuda-specification.md)
- [実装計画書](../docs/implementation-plan.md)
```

### 効率的な作成方法

1. **issues-summary.md**を開く
2. 各Issueの情報をコピー
3. GitHubで新しいIssueを作成
4. タイトル、本文、ラベル、マイルストーンを設定
5. 保存
