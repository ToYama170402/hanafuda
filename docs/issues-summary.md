# GitHub Issue作成ガイド

このドキュメントは、実装計画書に基づいてGitHub Issueを作成するためのガイドです。

## Issue作成の手順

### 1. メインIssueの準備

現在のIssue（仕様書に基づいて具体的な実装を計画する）をメインIssueとして、以下のSub Issueを関連付けます。

### 2. Sub Issueの作成

以下の30個のIssueをGitHubに作成し、メインIssueに関連付けます。各Issueには以下の情報を含めます：

- タイトル
- 説明（タスク内容）
- ラベル（enhancement）
- マイルストーン（該当フェーズ）
- 依存関係（Depends on #XX）

## ラベル定義

以下のラベルをGitHubリポジトリに追加することを推奨します：

- `setup` - プロジェクトセットアップ関連
- `core` - Core層の実装
- `domain` - Domain層の実装
- `facade` - Facade層の実装
- `yaku` - 役判定関連
- `koikoi` - こいこい機能
- `special-rule` - 特殊ルール
- `testing` - テスト関連
- `documentation` - ドキュメント
- `ci-cd` - CI/CD関連
- `performance` - パフォーマンス最適化
- `logging` - ログ機能
- `replay` - リプレイ機能
- `error-handling` - エラーハンドリング
- `configuration` - 設定・構成
- `events` - イベントシステム
- `optional` - オプション機能
- `priority-high` - 高優先度
- `priority-medium` - 中優先度
- `priority-low` - 低優先度

## マイルストーン定義

以下の4つのマイルストーンを作成します：

### M1: MVP完成
- 期限: プロジェクト開始から3週間
- 説明: 基本的なゲームフローが動作する最小限の機能

### M2: 完全ルール実装
- 期限: M1完了から2週間
- 説明: こいこいを含む完全なゲームルールの実装

### M3: 拡張機能実装
- 期限: M2完了から1週間
- 説明: カスタマイズ機能やイベントシステムの実装

### M4: リリース準備完了
- 期限: M3完了から1週間
- 説明: 最適化、ドキュメント、リプレイ機能の実装
